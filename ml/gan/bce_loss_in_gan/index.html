BCE Loss in GAN

목적

판별자는 BCE loss 값을 최소화 할려고 한다. 생산자는 BCE loss 값을 최대화 할려고 한다.

문제

Mode collapse

BCE loss 를 사용하면 생산자가 진짜 같은 가짜를 만드는데 목적을 둔다. 생산자가 0~9 까지의 클래스를 만드는게 목적이 아니기 때문에 mode collapse 가 일어나게된다. 그래서 우리는 0~9 까지를 골고로 생산하길 바라지만 진짜라고 판명난 몇 개의 숫자 클래스만 만들 가능성이 높다.

경사도 하강 사라짐 문제

훈련이 끝나가면 판별자는 0, 과 1에 가까운 값을 생산한다. 이전 전의 값도 0 과 1에 가깝기 때문에 변화값이 0에 가깝게 되고 이로 인해 학습이 느려진다.

참고

판별자를 훈련을 시킬 때 loss function에는 fake_img의 Y 의 값을 0 로 준다. 생산자를 훈련을 시킬 때 loss function에는 fake_img의 Y 의 값을 1 로 준다.

for real_X, real_Y in dataloader:
    disc_opt.zerograd()
    real_Y_pred = disc(real_X)

    fake_X = gen(noise())
    fake_Y_pred = disc(fake_X)
 # fake_img 의 Y 값을 0 으로 세팅
    fake_Y = zero_likes(fake_X)
    disc_loss = lossfun([real_Y_pred + fake_Y_pred], [real_Y + fake_Y])
    disc_loss.backward()
    disc.step()

    gen_opt.zerograd()

    fake_X_2 = gen(noise())
    fake_Y_pred_2 = disc(fake_X_2)
# [0, 1] 1이면 좋은거야 1이면 로스가 작아져야되
    gen_loss = lossfun(fake_Y_pred_2, ones_like(fake_Y_pred_2))
    gen.loss.backward()
    gen_opt.step()
